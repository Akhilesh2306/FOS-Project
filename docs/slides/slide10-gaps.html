ROADMAP

Phasing and Early Considerations

Phase 1 delivers the core sales intelligence platform. Phase 2 layers on Canva content generation. We have identified key items to address early in development to keep delivery smooth.

Phase 1 -- Sales Intelligence Platform
What we deliver: Proactive AI assistant that monitors deals, detects risks, generates recommendations, and delivers via Slack and web dashboard.

Detection Agent catches stale deals, stage changes, health drops
NBA Agent generates ranked actions with justifications
Playbook Agent creates 11-section text-based deal guides
RAG Agent searches battle cards and competitive intel
Feedback Agent collects thumbs up/down, tracks outcomes

Delivery: Slack (proactive + on-demand) and Web Dashboard (AE + Manager views)
Phase 2 -- Content Generation (Canva Layer)
What we add: Canva-native content generation on top of the base intelligence platform.

BrandKit API integration for on-brand proposal generation
Product Catalog ingestion so agents can recommend templates
Canva MCP tool added to agent runtime (tool #8)
Workspace Delivery so generated assets land in Canva
BrandBrain early adopter integration (if available)

Why this is additive: Architecture is modular. Adding a tool, a Bronze source, and an output format requires zero rework of Phase 1.
Items to Address Early in Development
Gong Connector Selection
Three options under evaluation: Fivetran, native HTTP/API, or Snowflake Data Cloud. Recommend finalizing early so Bronze layer development for 16 Gong objects can begin in parallel.
Content Pipeline Design
RAG pipeline needs chunking strategy, embedding configuration, and Cortex Search index design finalized. Planning to address in Sprint 1 alongside agent development.
MEDDPICC Field Validation
Architecture defines 28 custom fields for health scoring. Need to validate API names against Canva's actual Salesforce org to ensure schema alignment.
Content Source Confirmation
Architecture is designed for Guru as the knowledge base. Need to confirm this is what Canva uses so the content extraction pipeline targets the right system.
Gong Transcripts in RAG
Call transcripts are ingested to Bronze. Need to determine whether they should also be indexed in Cortex Search so agents can cite specific call quotes in recommendations.
Staleness Thresholds by Stage
Detection Agent uses configurable thresholds. Need Canva input on what "stale" means at each sales stage to tune alerts appropriately for their sales cycle.
Design Decisions for Phase 1
Area Phase 1 Approach Phase 2 Enhancement
CRM Updates Read-only. Agents do not write to Salesforce. Architecturally, humans should own CRM data for accuracy and trust. Explore assisted CRM population with AE approval step
Playbook Delivery Async generation. 11-section playbooks with RAG and LLM take 10 to 18 seconds. User gets a notification when the playbook is ready. Optimize with parallel section generation and caching
Manager View Pipeline overview with health scores, deal counts, and MEDDPICC completeness. Focused on visibility. Full coaching dashboard with rep activity patterns and deal review workflows
Notification Timing Polling-based detection every 60 minutes. Sufficient for deal-level cadence where actions are measured in hours and days. Add Salesforce Platform Events for near-real-time stage changes
Email Drafts Generate email text in Slack. AE copies and personalizes before sending. Keeps human in the loop. Direct integration with Gmail/Outlook for one-click draft creation
SE Handoff Agent recommends SE engagement. AE approves and triggers the notification. Ensures AE stays in control of the relationship. Auto-notify SE with opt-out, based on confidence threshold
