Latency Budget

Every flow in the system has a measurable end-to-end latency. Synchronous flows complete in seconds. Proactive detection flows take up to 85 minutes because they depend on polling intervals. Dashboard reads are near-instant because all data is pre-computed. The table below summarizes every flow, its type, and the component that dominates wall-clock time.
Flow Type End-to-End Latency Dominant Factor
1. Stalled Deal Detection Proactive 45 to 85 min slow Detection Agent 60-min scan interval
2. Meeting Prep Synchronous 3 to 6 sec fast LLM inference (2 to 4 sec)
3. Stage Change Playbook Event-driven 45 to 85 min + 12 to 20 sec slow Detection scan + LLM inference for 11 sections
4. SE Technical Handoff Detection + Human 2 to 4 hours variable Gong processing + sync + AE approval time
5. Manager Dashboard Read-only <500 ms fast Pre-computed data, Snowflake query time
Key Insight
  • Synchronous flows are fast (3 to 6 seconds) because they bypass the Detection Agent and go directly to an agent.
  • Proactive flows are slower (45 to 85 minutes) because the 60-minute Detection Agent scan interval dominates wall-clock time.
  • Dashboard reads are near-instant (<500 ms) because they consume pre-computed data with no agent execution.
  • The LLM is the single largest latency contributor in every flow that generates text. Optimizing LLM inference (model selection, prompt engineering, caching) has the highest return on latency improvement.

Architectural Invariants

Five patterns repeat across all five flows. These are not guidelines or suggestions. They are structural invariants that every component in the system respects. Violating any of these patterns means the data trace breaks.
1
Semantic Layer = Single Query Interface
Every flow that reads deal data goes through the Semantic layer. The Detection Agent queries Semantic views to find triggers. The NBA Agent queries v_opportunity_llm_context for deal context. The Dashboard queries v_pipeline_overview and v_deal_health_summary. No component queries Bronze, Silver, or Gold directly. The Semantic layer is the single source of truth for all consumers.
2
Event Hubs = Coordination Bus
All agent-to-agent communication flows through Azure Event Hubs. Detection publishes, NBA and Playbook consume. Agents publish notifications, Slack Bot consumes. This decouples producers from consumers. Adding a new consumer (e.g., a Microsoft Teams bot) means subscribing to the same topics. No existing component changes.
3
Hybrid Tables = Write Target
Every agent write goes to a Hybrid Table in the Bronze schema: fos_agent_events, fos_next_best_actions, fos_playbooks, fos_feedback_log, fos_audit_log. Hybrid Tables provide ACID writes with millisecond latency. These tables feed back into the medallion pipeline (Dynamic Tables refresh when Hybrid Tables change), creating the intentional write-back loop.
4
Cortex LLM = Generation Engine
Any flow that produces natural language output (NBA recommendations, meeting prep briefs, playbooks, SE briefing packages) uses Cortex LLM for generation. Data never leaves the Snowflake security perimeter. The LLM is always the latency bottleneck (2 to 15 seconds depending on output size), but it is also where the core value is created.
5
Degraded-but-Functional Fallback
Every flow has a fallback when the LLM or a search service is unavailable. The fallback is always "return the structured data from the Semantic layer without LLM synthesis." For example, if Cortex LLM times out during meeting prep, the AE still gets the raw deal context, MEDDPICC scores, and stakeholder list. Less polished but still useful.

Open Questions

These are unresolved design questions that affect the latency, reliability, or scope of one or more flows. Each question needs research and a decision before the corresponding flow can be considered production-ready.
Open Questions
  • Can detection go to 15 to 30 minutes? The current 60-minute scan interval dominates proactive flow latency. Reducing to 15 or 30 minutes cuts worst-case latency in half but increases compute cost. What is the cost per scan for a 1,000-deal pipeline? For 10,000 deals?
  • Can Salesforce Platform Events supplement polling? Real-time triggers from Salesforce (via Platform Events or Change Data Capture) pushed to Event Hubs would reduce proactive flow latency from 75 minutes to seconds. This would fundamentally change Flows 1 and 3.
  • Parallel vs. sequential playbook sections? The 11-section playbook is a large LLM generation (2,000 to 5,000 tokens). Should sections be generated in parallel (multiple smaller LLM calls) or sequentially (one large prompt)? What is the quality vs. latency tradeoff?
  • Notification deduplication across channels? If a manager views an at-risk deal on the dashboard (Flow 5) and the AE gets a Slack notification about the same deal (Flow 1), is there coordination? Should the manager's view show "AE was notified at 2:34 PM"?
  • Fallback LLM? If Cortex LLM is unavailable, do we have a secondary path? Routing to an external LLM (e.g., direct Anthropic API) is an option, but data would leave the Snowflake security perimeter temporarily. Is that acceptable?
